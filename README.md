# Workflow Intake Form

A professional and elegant intake form to collect information about users' workflows. The form securely stores data in Supabase via a server-side proxy and is designed to be deployed on Render.

## Features

- Professional, modern design with gradient backgrounds
- Inter font for enhanced readability
- Subtle animations and hover effects
- Form validation
- Secure submission via server-side proxy to Supabase
- Loading states and user feedback
- Mobile-friendly responsive layout

## Setup Instructions

### Prerequisites

- Node.js installed (for local development)
- Supabase account (sign up at [supabase.com](https://supabase.com))

### 1. Local Development

1. Install dependencies:
   ```bash
   npm install
   ```

2. Copy the environment example:
   ```bash
   cp .env.example .env
   ```
   
3. Update `.env` with your Supabase credentials:
   ```bash
   SUPABASE_URL=your_supabase_project_url
   SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
   ```

4. Start the development server:
   ```bash
   npm start
   ```

### 2. Configure Supabase

1. Create a new Supabase project
2. Create a table named `intake_submissions` with the following schema:

```sql
CREATE TABLE intake_submissions (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name TEXT NOT NULL,
  email TEXT NOT NULL,
  workflow TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

3. Get your Supabase URL and Service Role Key (found in Project Settings → API)

### 3. Deployment to Render

To deploy on Render as a Web Service:

1. Push this code to a GitHub repository
2. Connect your GitHub repo to Render
3. Create a new Web Service
4. Set the Root Directory to your repo root
5. Use `npm start` as the start command
6. Add your environment variables in Render dashboard:
   - `SUPABASE_URL` 
   - `SUPABASE_SERVICE_ROLE_KEY`
7. Your form will be live at your Render URL

### Environment Variables

The application requires these environment variables:
- `SUPABASE_URL` - Your Supabase project URL
- `SUPABASE_SERVICE_ROLE_KEY` - Your Supabase service role key for secure server access

## Form Fields

The intake form collects:

1. Name (text input)
2. Email (email input)
3. Workflow Description (textarea)

## Data Storage

Submitted data is stored in the `intake_submissions` table in Supabase with:
- Unique ID
- Name
- Email
- Workflow description
- Timestamp of submission

## Security Approach

This implementation uses a secure server-side proxy approach:
- Database credentials remain secure on the server
- Supabase service role keys are never exposed to clients
- All form submissions go through the `/api/submit-intake` endpoint
- Additional validation and sanitization happen on the server
- CORS policies are controlled by your server

## Notification System

To receive notifications when someone submits the form via Discord:

1. Create a Discord webhook in your server:
   - Go to your Discord Server Settings
   - Navigate to Integrations -> Webhooks
   - Create a new webhook and copy the URL
   - Or click on an existing webhook and copy its URL

2. Update your environment variables:
   - `DISCORD_WEBHOOK_URL` - Your Discord webhook URL

Once configured, you'll receive detailed notifications in your chosen Discord channel every time someone submits the form, including the submitter's name, email, workflow description, and submission timestamp.

## Admin Dashboard

The application includes a secure admin dashboard to view workflow submissions:

1. Access the dashboard at `/admin` route
2. Login with your admin credentials:
   - Default username: `admin` (can be changed via `ADMIN_USERNAME` env var)
   - Default password: `admin123` (can be changed via `ADMIN_PASSWORD` env var)
   - For enhanced security, set `HASHED_ADMIN_PASSWORD` to a bcrypt hash of your password

### Admin Features

- View all workflow submissions in a table format
- See submission details (ID, name, email, workflow, date)
- Click on truncated workflow descriptions to view the full text
- Track submission statistics (total submissions, today's submissions)
- Secure session-based authentication using Supabase credentials

### Admin Database Setup

To use Supabase authentication, you need to create an `admin_users` table in Supabase:

```sql
CREATE TABLE admin_users (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  email TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  role TEXT NOT NULL DEFAULT 'admin', -- 'admin' or 'super_admin'
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

To add an admin user, insert a record with a bcrypt-hashed password:

```sql
INSERT INTO admin_users (email, password_hash, role) 
VALUES ('your-email@example.com', '$2b$10$lkjasdnflkjandsf...', 'admin');
```

You can generate a bcrypt hash using online tools or Node.js bcrypt library.

### Securing Admin Access

For production environments, we strongly recommend:

1. Creating admin users in the Supabase admin_users table
2. Using strong passwords for admin accounts
3. Setting the appropriate role ('admin' or 'super_admin')
4. Enabling SSL/HTTPS for encrypted connections
5. Consider restricting IP addresses that can access the /admin route

### Environment Variables for Admin Access

- `SESSION_SECRET` - Secret for session encryption (important to change from default!)
- `SUPABASE_URL` - Your Supabase project URL
- `SUPABASE_SERVICE_ROLE_KEY` - Your Supabase service role key for server access
